## How to Clone/Copy a Template

1. **Copy the template directory:**
   ```bash
   cp -r templates/web-app my-dapp
   cd my-dapp
   ```

2. **Install dependencies:**
   ```bash
   pnpm install  # or npm install
   ```

3. **Configure environment:**
   ```bash
   cp .env.example .env.local
   # Edit .env.local with your settings
   ```

4. **Start developing:**
   ```bash
   pnpm dev  # or npm run dev
   ```

## How to Wire RPC URLs and Keys

### Environment Variables

All templates use environment variables for configuration:

**Web/Mobile Apps:**
```env
NEXT_PUBLIC_DEMIURGE_RPC_URL=http://127.0.0.1:8545/rpc
NEXT_PUBLIC_ABYSS_GATEWAY_URL=http://localhost:4000/graphql
```

**Rust/Node:**
```env
DEMIURGE_RPC_URL=http://127.0.0.1:8545/rpc
ABYSS_GATEWAY_URL=http://localhost:4000/graphql
```

### SDK Initialization

**TypeScript:**
```typescript
import { DemiurgeSDK } from '@demiurge/ts-sdk';

const sdk = new DemiurgeSDK({
  rpcUrl: process.env.NEXT_PUBLIC_DEMIURGE_RPC_URL || 'http://127.0.0.1:8545/rpc',
});
```

**Rust:**
```rust
use demiurge_rust_sdk::DemiurgeSDK;

let rpc_url = env::var("DEMIURGE_RPC_URL")
    .unwrap_or_else(|_| "http://127.0.0.1:8545/rpc".to_string());
let sdk = DemiurgeSDK::new(&rpc_url);
```

### Private Keys

**Never commit private keys to version control!**

Store keys securely:
- **Browser/Web**: localStorage (for development)
- **Server**: Environment variables or secure key management
- **Mobile**: Secure storage (Keychain/Keystore)

Example:
```typescript
// Load from localStorage (web)
const privateKey = localStorage.getItem('demiurge_private_key');

// Or from environment (server)
const privateKey = process.env.DEMIURGE_PRIVATE_KEY;
```

## Deployment

### Web Apps (Vercel/Netlify)

```bash
pnpm build
# Deploy the `out` or `.next` directory
```

### Mobile Apps

```bash
# Build for production
npx expo build:android
npx expo build:ios
```

### Rust Services

```bash
cargo build --release
# Deploy the binary
```
